## GDPR Compliance Checklist

| Section Title | Result |
| :--- | :--- |
| **Personal data mapping and minimization** | |
| Have all personal data collected and processed been identified? | ✅ PASS |
| Have you ensured that only necessary personal data is collected? | ✅ PASS |
| Is user age recorded to verify that the booker is over 15 years old? | ✅ PASS |
| **User registration and management** | |
| Does the registration form include GDPR-compliant consent? | ❌ FAIL |
| Can users view, edit, and delete their own personal data? | ✅ PASS |
| Is there a mechanism for the administrator to delete a reserver? | ✅ PASS |
| Is underage registration (under 15 years) and booking restricted? | ✅ PASS |
| **Booking visibility** | |
| Are bookings visible to non-logged-in users only at resource level (no personal data)? | ✅ PASS |
| Is personal data hidden from unauthorized users? | ❌ FAIL |
| **Access control and authorization** | |
| Have you ensured that only administrators can add, modify, and delete resources and bookings? | ✅ PASS |
| Is the system using role-based access control? | ✅ PASS |
| Are administrator privileges limited and not exposed? | ❌ FAIL |
| **Privacy by Design principles** | |
| Is Privacy by Default implemented? | ✅ PASS |
| Are logs implemented without excessive personal data? | ✅ PASS |
| Are forms and system components designed with data protection in mind? | ❌ FAIL |
| **Data security** | |
| Are CSRF, XSS, and SQL injection protections implemented? | ❌ FAIL |
| Are passwords securely hashed with a strong algorithm? | ✅ PASS |
| Are data backup and recovery processes GDPR-compliant? | ⚠️ ATTENTION |
| Is personal data stored in data centers located within the EU? | ⚠️ ATTENTION |
| **Data anonymization and pseudonymization** | |
| Is pseudonymization used where appropriate? | ❌ FAIL |
| **Data subject rights** | |
| Can users download or request all personal data related to them? | ❌ FAIL |
| Is there an interface or process for users to request the deletion of their personal data? | ❌ fail |
| Can users withdraw their consent for data processing? | ❌ FAIL |
| **Documentation and communication** | |
| Is there a privacy policy available to users during registration and easily accessible? | ❌ FAIL |
| Are administrators and developers provided with documented data protection practices...? | ❌ FAIL |
| Is there a documented data breach response process...? | ❌ FAIL |
